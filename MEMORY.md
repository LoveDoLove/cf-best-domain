# MEMORY.md

仓库长期记忆（AI Memory）说明：用于存放用户偏好、项目目标与持久约定，供 AI Assistant 在对话中恢复上下文与行为决策。

存放与目录结构：

- `MEMORY.md` — 长期记忆主文件，包含：
  - 用户偏好（沟通风格、语言偏好、默认命令习惯等）
  - 项目目标与约定（代码风格、部署目标、重要里程碑）
  - 持久模板（常用对话启动语、CI/CD 约定）

- `memory/tasks.md` — 跨对话任务追踪（待办 / 进行中 / 完成），供 Agent 与用户同步长期任务进度。

- `memory/YYYY-MM-DD.md` — 每日日志，记录当天 AI 的工作记录、决策要点与备注，格式为 Markdown。

示例条目：

```
- 用户偏好:
  - 语言: 中文（简体）
  - 风格: 简洁、偏工程师口吻

- 项目目标:
  - 优化域名收集脚本，确保隐私合规
```

使用规范：
1. 每次会话开始，Assistant 应优先读取 `MEMORY.md`，然后基于需要读取 `memory/tasks.md` 或当日日志恢复工作上下文。
2. 当 Agent 执行会改变长期约定的操作（例如：变更编码约定或 CI 流程），必须在 `MEMORY.md` 中记录变更并写入变更记录与时间戳。
3. `memory/tasks.md` 用于记录跨会话待办，Agent 在每次任务状态更新时应写入该文件以保持同步。

隐私与审计：
- 禁止将任何敏感信息（API key、密码、个人身份信息）写入 `MEMORY.md` 或 `memory/` 目录下的任何文件。
- 所有更改应保留作者/Agent 标识与时间戳，以便审计。

技能包（.agents/skills）索引：
- 建议在 `MEMORY.md` 中维护一个简短索引，列出已安装技能包的名称、来源（GitHub URL）与最新 commit/hash。

快速上手示例：

```
请读取 AGENTS.md 和 MEMORY.md，恢复你的身份和工作状态，然后继续我们的工作。
```
